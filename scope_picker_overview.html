Scope Picker

A self-contained browser tool that lets a solution architect take the master reference architecture and carve out a precise, project-specific scope — ready to land in BlueDolphin.

Single HTML file — no install needed Fully file-driven — works offline 3-step guided workflow Designed to become a Navigator tab
⚠️

Scope Picker is currently a Proof of Concept (POC). It works fully as a standalone tool. The final step — applying the scope back into BlueDolphin — has been built and tested with a dry-run, but has not yet been executed live against a real project. Always run a dry-run first before a live apply.

01 — The Big Picture

What problem does Scope Picker solve?

The master architecture in BlueDolphin covers everything — all products, all customer segments, all channels, all E2E use cases. For any given customer project, only a relevant slice of that master applies. Today, identifying and documenting that slice is manual and time-consuming. Scope Picker makes it structured, visual, and exportable in minutes.

🗺️

Without Scope Picker

  • Manually browsing the master model to find relevant items
  • No structured way to record which E2E use cases apply
  • Requirements are mapped informally, often in spreadsheets
  • Gaps and missed coverage only discovered late
  • Re-creating objects in BlueDolphin for each project by hand
✅

With Scope Picker

  • Guided 3-step flow — set context, map requirements, review
  • Upload your requirements file and map each one visually
  • Coverage gaps surfaced automatically with bar charts
  • Author brand-new objects (not in master) inline, in context
  • Export one file that drives the BlueDolphin setup automatically

03 — What You Need

Pre-requisites

Two files and a browser are all you need to run the Scope Picker. The BlueDolphin step only comes later, once you are happy with the scope you have defined.

📦

e2e_data.json

The extended master model exported from BlueDolphin. Must include a reference_data block covering Product Categories, Customer Segments, Channels, and Application Components. This is extracted once using the extract_e2e_data_v3.py script.

📊

Requirements file

Your customer's project requirements as an .xlsx or .csv file. The tool auto-detects the header row, the statement column, and any ID column. The current reference file has approximately 1,463 requirements.

🌐

A modern browser

Open scope_picker.html directly in Chrome, Edge, or any modern browser. No server, no install, no login required. Everything runs locally on your machine.

04 — The Workflow

Three steps to a scoped project

Every session follows the same guided flow. You can save your progress at any point and pick up where you left off.

1
Step 1
Project Setup — Define the context
Start by telling the tool which slice of the master architecture applies to this customer. You select from four shortlists that have been pre-populated from the master model. Your selections here become the constrained pick-lists you will use when mapping individual requirements in Step 2 — so accuracy matters.
For each dimension, you can also add items that do not yet exist in the master using the inline + New button. These get a pink "NEW" badge and are automatically included in your scope.
🟠 Product Categories 🩷 Customer Segments 🔵 Channels 🩵 Application Components ✦ + New (for non-master items)
2
Step 2
Requirements Mapping — Connect requirements to the architecture
Upload your requirements file. The tool displays each requirement as a row in an editable grid. Click Map on any row to open a side drawer where you connect that requirement to the architecture:
  • 1. Tag the requirement with the applicable Channel, Product Category, Customer Segment and Application Component from your Step 1 shortlist.
  • 2. Select one or more E2E Use Cases that this requirement falls under.
  • 3. Under each E2E, tick the Business Services it touches — and whether they are Core (always needed) or Conditional (only in certain contexts).
You can select multiple rows and bulk map them at once. You can also save your progress and reload it in a later session. And if a requirement points to something that does not exist in the master yet, you can author it right here in context — without leaving the drawer.
🟢 Core Business Services 🟡 Conditional Business Services ✦ Author new E2Es / Services inline 💾 Save & resume supported
3
Step 3
Review & Generate — Verify, check coverage, export
Three sub-tabs let you review your work from different angles before exporting.
🧭

Scope Navigator

A read-only walk of the scoped model — exactly as it will appear in BlueDolphin. Browse by Customer Journey (or Value Stream). Click an E2E to see its Business Services and the individual requirements mapped to each.

📊

Coverage Stats

Bar charts showing how many items from each of your four shortlists (Products, Segments, Channels, App Components) are actually covered by requirements. Click any bar to drill into which E2Es and requirements cover it. Gaps are highlighted.

⬇️

Generate

Validation checks catch any errors before you export. Review scorecards and project context. Then download the scope_instruction.json file — or preview it inline before saving.

05 — Authoring New Content

What if something isn't in the master model yet?

Customer projects often have requirements that touch capabilities not yet defined in the master architecture. Scope Picker lets you author these new objects inline — without losing your context or switching to another tool. Everything you create gets tagged as "new" and will be created fresh in BlueDolphin when you apply.

🟠 New reference items NEW badge

In Step 1, each pick card has a + New button. Type a name, press enter — the item is added to your shortlist and automatically selected. Useful for project-specific Product Categories, Segments, Channels, or Application Components not in the master.

🔷 New E2E Use Case

In the Step 2 mapping drawer, click + Create new E2E Use Case. Fill in the full BOEM form: Title, Job To Be Done, Business Outcome, what the scope owns and what it doesn't, and a description. Assign it to a Value Stream and Customer Journey.

🟢 New Business Service

Under any E2E in the drawer, click + Add new Business Service with the full BOEM form. Set it as Core or Conditional and optionally add a condition note. You can also attach an existing Business Service from the master to a new E2E.

🔄 New Business Process variant

Under any Business Service, add a process variant inline. Variants automatically inherit the Channel, Product, and Segment you tagged at the top of the drawer — so the context is never lost. The first process on a new service is always the base flow.

06 — The Output

What Scope Picker produces

The tool generates a single structured export file — scope_instruction.json — that captures every decision you made. This file is the bridge between your scoping session and BlueDolphin.

📄 scope_instruction.json — what's inside
🏷️
Project details
Project name, prefix (used to name objects in BlueDolphin e.g. [MyProject] Order Management), and the source workspace to clone from.
🗂️
Scope — the selected architecture objects
The full list of E2E Use Cases, Business Services, Business Processes, and Application Services that were selected during mapping. Each object is tagged as either master (to be cloned) or new (to be created).
📋
Requirements
Every in-scope requirement with its full statement, ID, and all of its architecture mappings (which E2E, which Business Services, which Channel/Product/Segment).
✦
New objects block
All the objects you authored during the session that don't exist in the master — new reference items, new E2E Use Cases, new Business Services, and new Business Process variants — with all their details ready to be created in BlueDolphin.
⚙️

What happens next — apply_scope.py

The apply_scope.py script reads the export file and writes everything into BlueDolphin automatically:

  • Master objects are cloned into your project workspace with the [Project] name prefix, carrying all their BOEM descriptions.
  • New objects you authored are created from scratch.
  • Relationships between all objects are recreated — but only within the project copies, never touching the master.
  • Requirements become Requirement objects in BlueDolphin, linked to the E2E Use Cases and Business Services they were mapped to.
  • Running it again is safe — it will not create duplicates (it matches by name).

08 — What's Next

Open items & planned improvements

The tool is functional and ready for use. These are the known open items and agreed next steps.

  • 1 Run apply_scope.py --dry-run against a small scope to confirm the "Requirement" object type name matches what is configured in BlueDolphin — then run a single-requirement live test before doing a full apply.
  • 2 Optionally include Application Components in the save/load progress feature and the Step 3 review panel for a more complete round-trip.
  • 3 Upgrade the Coverage Stats charts to ECharts for richer interactivity — the current charts are custom-built and functional, but ECharts would allow more advanced visualisations if needed.
  • 4 Application Service filtering (showing which Application Services sit under each Application Component) is deferred until the corresponding links exist in BlueDolphin.
  • 5 The tool is designed to eventually drop into the E2E Navigator as a new tab, eliminating the need to run it as a separate standalone file.
