The Challenge

BSS transformation projects are large, complex, and high-stakes — yet the scoping process has stayed manual and fragmented.

Spreadsheet dependency

Scope driven by a massive, manually maintained Excel file — version-controlled by email, understood by few.

No requirements traceability

No structured link between customer requirements and the E2E architecture components being scoped.

Inconsistent outputs

Each architect produces different scope documents, making quality control and peer review impossible at scale.

Generic workshop materials

Customer workshops use generic architecture diagrams, not project-specific views — reducing engagement and accuracy.

Knowledge locked in individuals

Scope decisions live in architects' heads and email chains, not in a reusable, auditable system.

Slow iteration cycles

A single scope review cycle can take days of manual work, slowing pre-sales and delivery rhythm.

The Vision

"A connected, automated pipeline that transforms raw project requirements into customer-ready architectural views — accurately, consistently, and in hours rather than days."

Spec-Sync
Tool 1 of 4 · Requirements Pipeline

What it is

›Industry-standard synchronization tool connecting BDD (Gherkin) feature files with Azure DevOps and Jira
›Creates and maintains Test Cases and Requirement work items directly from human-readable feature files
›Tag-based linking (@req:ID) ties each scenario to architecture and test work items

Role in scoping

›Imports requirements into Azure DevOps / Jira as structured work items with stable IDs
›Ensures every requirement is traceable before entering the scoping pipeline
›Produces the XLSX / CSV that Scope Studio ingests in Step 2
›Runs per module / domain — separate config per functional area

BlueDolphin
Tool 3 of 4 · Architecture Repository

Single source of truth for the E2E architecture — master model and all project-specific scoped views

What BlueDolphin provides

›Enterprise architecture platform by BiZZdesign — purpose-built for managing architecture at scale
›Stores the [MASTER] workspace with all E2E Use Cases, Business Services, and Business Processes
›Each project gets its own [ProjectName]-prefixed workspace with cloned, scoped objects
›Requirements linked to architecture via Realization relations
›OData REST API lets Scope Studio extract master data and Navigator read project scopes

Key scripts

extract_e2e_data_v3.py

Pulls full master model via OData → e2e_data.json for Scope Studio

apply_scope.py

Reads scope_instruction.json, calls BlueDolphin REST API, clones objects into project workspace

E2E Solutions Navigator
Tool 4 of 4 · Workshop Engine

HTML-based · Reads BlueDolphin via OData · Customer workshop-ready · Designed to become the unified front-end for the entire toolchain

What the Navigator does

›Connects to BlueDolphin via OData API to retrieve the project-scoped workspace
›Renders scoped architecture in an interactive, clean HTML interface
›Enables consultants to walk customers through their specific project scope — not generic diagrams
›Accessible to non-technical stakeholders — navigable, self-explanatory
›Planned: Scope Studio embedded as a tab within Navigator for a unified experience
